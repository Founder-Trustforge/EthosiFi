# Security Policy

## Supported Versions

We currently provide security updates only for the latest released version.

## Reporting a Vulnerability

**We take security seriously.**

If you believe you have found a security vulnerability in EthosiFi, please report it responsibly.

**Preferred method:**
- Open a **private** GitHub issue in this repository  
  Title format: `[Security] Vulnerability report â€“ [short description]`

**Alternative (if you prefer):**
- Email: security@ethosifi.eth (if you have set up this address)

**What to include in your report:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)
- Any proof-of-concept code (please do **not** include exploit code in public issues)

We will acknowledge receipt within 48 hours and aim to provide an update (including triage and next steps) within 7 days.

Thank you for helping keep the EthosiFi protocol and community safer.
